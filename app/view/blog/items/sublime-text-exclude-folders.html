Sublime text: exclude folders from side bar

When developing Node JS applications, you alse get a node_modules directory. This direcotry contains all Node JS modules on which your project is depending.

The problem I keep having is that, because the node_modules directory contains a very large number of files, Sublime Text will also index them. This takes up a lot of my processor capacity and leaves me with a burning laptop.

Another thing is that the Goto anything (CTRL-P) will also search the node_modules directory, which in any case you just don't want. This takes up a lot of time and search results will not be clean.

Solution

Fortunately, Sublime text has a way to get past this. We can configure which directories to exclude from Sublime text.

First of all we have to get the existing settings from the default settings. To do this, in Sublime go to Preferences, Settings - Default.

There copy the folder_exclude_patterns, which will look something like:

"folder_exclude_patterns": [".svn", ".git", ".hg", "CVS"],

After this, we have to copy this to the user settings to override this. In Sublime go to Preferences, Settings - User and add the folder_exclude_patterns. Add the node_modules to the array, leaving it to look like this:

"folder_exclude_patterns": ["node_modules", ".svn", ".git", ".hg", "CVS"],

This fix will result in node_modules directory not showing up in the side bar and search results
